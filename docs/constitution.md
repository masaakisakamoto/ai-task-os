# AI Task OS Constitution

## Purpose
We are building an AI-native Execution OS, not a task manager.

## Core Concept
Task → Run → Eval → Memory

## Positioning
- Not a productivity tool
- Execution infrastructure for AI builders

## Strategy
- OSS first
- Hosted later
- Developer-first

## Non-goals
- Do NOT build a generic task manager
- Do NOT start with UI
- Do NOT include memory engine in v0.1

## Design Principles
- Contract first
- Reproducibility over convenience
- Deterministic before AI

## AI Usage Rules
- AI can write code
- AI cannot make final decisions
- AI operates in branch/PR only

## Quality Bar
- Must run in 10 minutes
- Must demonstrate value in 1 example
