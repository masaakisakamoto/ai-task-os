# Acceptance Checklist

## Scope
- Did the change stay within task scope?
- Did it avoid non-scope changes?
- Did it avoid hidden refactors?

## Product Drift
- Is this still not a task manager?
- Is it still developer-first?
- Is it still CLI-first?
- Is v0.1 still minimal?
- Was no UI introduced?
- Was no memory engine introduced?

## Code Quality
- Is the change small and understandable?
- Are error messages reasonable?
- Is behavior deterministic?
- Were unnecessary dependencies avoided?

## Verification
- Were the requested checks run?
- Did the success criteria pass?
- Did existing behavior remain intact?

## Decision
- Approve
- Approve with small fixes
- Reject
